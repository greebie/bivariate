What Does it Do?

Although still in development, this project will let you store a selection of "committees" to your account and then you can use these committees to produce a visualisation that will compare those committees based on common membership.

If you are logged in, you can select your own committees, but if you just want to play, you can use my small selection of test data. At this stage I am looking to improve the user interface, but you can simply select a "group" (category of committees) which will add some committees to the screen. When you click on each of these committees, it will add them to the visualisation graph.

After one click, not much will happen, but after two, you should see a Venn Diagram representing the two committees and the number of "members" they have in common. A similar graph will show for three committees.

After four clicks, assuming that there are sufficient members to produce one, a correspondence graph will be created to show the interrelation of the committees based on their memberships, using the top two dimensions from a correspondence analysis. While a detailed description of correspondence analysis is not possible here, you can learn more by reading my tutorial at the Programming Historian or the articles in the Bibliography below.

What is it For?

While the main reason I have developed this was "because I can," I do have a few ideas about what it could be used for.

This demo builds on a tutorial I provide at Programming Historian, and has been inspired by a number of scholars whose writing has helped learn both the power and potential of bivariate network analysis in social science research. These include:

  • Dr. Katherine Faust

For one, it could help students learn more about correspondence analysis and what it does. While the current literature is full of Greek letters and useful examples, this project can show correspondence analysis in real time. Rather than reading about bivariate networks in detail, they can simply create them on the fly, using their own data and gain a better understanding of what the analysis is actually about.

While bivariate analysis tends to be exploratory, the advantage of this tool is that is enables the adding and removing of committees on the fly. This can be important when one committee is so different that it confounds the plot and makes it hard to see what is happening. Of course, this is a bad way to approach towards hypothesis testing, but as an exploratory tool it can help narrow the scope of an analysis, making stronger research questions.

Finally, i discovered while working with very large Web Archives (500GB or bigger), more attention is needed for exploratory data analysis, for both its strengths and for its weaknesses. While much Big Data analysis is focussed on predictions based on correlations, less work has been done on how to explore a system to understand it. This path has great potential, but also a large number of pitfalls. In order to take advantage of the potential and minimize the effect of the pitfalls, we need strong tools. This particular tool may not be the answer, to this more general problem but hopefully it can contribute to a larger body of work.

What is Your Policy on Privacy?

This project is a demonstration for personal use only and I offer no warrantees on the security or privacy of the data you produce here. If you have sensitive data that you want to protect, I would not use this service. However, you can Contact Me if you are interested in using my set up on top of a database you control yourself. You are also welcome to use the source code as licensed.

This project uses Firebase to do the heavy database work. The positive is that it is fast. It is also (mostly) private. But Google will use it to improve their algorithms, so if that offends you, do not login.
